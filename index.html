页面权限管理

方案一：

前端路由信息完全血丝，根据后端返回值动态生成页面菜单。前端在router.js中定义好所有的路由。为了防止用户输入URL访问，需要在跳转时添加路由守卫router.beforeEach，对权限进行判断，不满足权限跳到401，404页面

方案二：

后端返回路由权限名，前端存储完整路由权限表。用户登录后通过Token获取用户role，跟根据用户role算出拥有权限的路由，并通过router.addRoutes动态挂载路由。

  • 路由分为两种：constantRoutes（静态路由）与asyncRoute（动态路由）
  • 使用addRoutes方法动态添加路由

方案三：

后端返回完整异步路由表

按钮权限管理

方案一

在获取到用户的role后，用v-if判断或者封装自定义指令v-auth

const auth = Vue.directive('auth', {
    inserted: function(el, binding, vnode) {
        const actionName = binding.value || '';
        const permissions = store.getters['user/permissions'];
        const [requiredPermission, requiredAction] = actionName.split(.)
        if(
            !(
                permissions[requiredPermission] &&
                requiredAction &&
                permissions[requiredPermission].includes(requiredAction)
            ))
            {
                (el.parentNode && el.parentNode.removeChild(el)) || (el.style.display = 'nones')
            }
    }
})

TRACE（HTTP 1.1）

  • 追踪路径
    TRACE 方法是让 Web 服务器端将之前的请求通信环回给客户端的方法。发送请求时，在 Max-Forwards 首部字段中填入数值，每经过一个服 务器端就将该数字减 1，当数值刚好减到 0 时，就停止继续传输，最 后接收到请求的服务器端则返回状态码 200 OK 的响应。客户端通过 TRACE 方法可以查询发送出去的请求是怎样被加工修改 / 篡改的。这是因为，请求想要连接到源目标服务器可能会通过代理 中转，TRACE 方法就是用来确认连接过程中发生的一系列操作

持久连接节省通信量

Keep-Alive

持久连接的好处在于减少了 TCP 连接的重复建立和断开所造成的额 外开销，减轻了服务器端的负载。另外，减少开销的那部分时间，使 HTTP 请求和响应能够更早地结束，这样 Web 页面的显示速度也就相 应提高了。在HTTP/1.1 中，所有的连接默认都是持久连接

Pipeline技术

持久连接使得多数请求以管线化(pipelining)方式发送成为可能。从 前发送请求后需等待并收到响应，才能发送下一个请求。管线化技术 出现后，不用等待响应亦可直接发送下一个请求。

高维多元数据

  • 高维多元数据是指每个数据对象有两个及以上独立(高维)或相关属性（多元）的数据
  • 常规二维三维数据可以采用常规可视化方法表示:散点图（Scatter Plot），当超过三维时可可以使用视觉编码（颜色，大小，形状）

三类方法

  • 空间映射
  • 图标法
  • 基于像素可视化方法

散点图矩阵

对于N维数据，采用$N^2$个散点图逐一表示N个属性之间的两两关系，组成$N*N$,位于对角线上的属性可用于揭示数据在特定属性上的分布
